9总结

通过SpringMVC的学习我们可以了解到SpringMVC的一些配置，怎么去整合Spring，Mybatis，还有就是dao层对数据库的访问怎么变得更加优化。

看看我以前使用的多层结构，没有使用配置文件来实现一个SSM项目的时候是怎样的

项目结构图如下：

我们来分析一下老版本的SSM的整个结构，前端页面通过post请求方式访问Controller中的具体方法，

Controller通过反射并调用具体的方法，然后在Controller中，通过Service层访问dao。

例如一个登录过程： Controller层去调用service层的方法，而service 的具体实现是ServiceImpl,

ServiceImpl层调用的是dao层的方法，而dao层具体的实现是daoImpl.

UserService userService=new UserServiceImpl();

userService.findById(str[0]);

private UserDao userDao=new UserDaoMysqlImpl();

public User findById(String id) throws Exception {
return userDao.selectById(id);
}

具体的操作可以去百度一下JDBC访问数据库的方式。
